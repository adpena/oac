Open Agent Capsule (OAC)

Specification and toolkit for portable agent state.

Open Agent Capsule is a file-first format for maintaining agent identity, persona, procedures, and memory as a versioned, portable canonical artifact.

System Utility

1. Decoupled State and Execution

OAC separates the agent's core instructions and memory from the specific execution harness. This enables the migration of an agent's state between different models and tools without data loss or manual re-prompting.

2. Verified Intelligence Supply Chain

OAC implements a cryptographic signing path (HMAC-SHA256 locally, with SSH-Ed25519 support for release distribution). Updates to an agent's state must pass structural evaluation gates and be signed before they are promoted. This ensures that the agent's behavior is traceable to a verified source.

3. Targeted Context Selection

Using structural indexing and selective hydration, OAC allows agents to load only the specific records required for a task. This reduces data processing requirements and improves the precision of agent responses.

Architecture

  • Canonical Layer: Markdown/YAML/JSON source of truth.
  • Adapter Layer: Bidirectional compilers for Codex, Claude Code, Gemini, OpenClaw, and OpenCode.
  • Orchestration Layer: Model Context Protocol (MCP) server for record discovery and learning loops.
  • Distribution Layer: Deterministic snapshots with polymorphic signatures.
